# TODO mode

プロジェクトルートの `TODO.md` を読み、todo mode に入る。

## 起動時

1. TODO.md を Read する
2. 各タスクの状態を報告する
3. `[-]` のタスクがあれば「他プロセスが作業中」と伝える
4. 上から順に最初の `[ ]` タスクに着手する（確認不要）

## チェックボックスの状態

- `[ ]` — 未着手
- `[-]` — 作業中（他プロセスが着手済み。重複作業を避けること）
- `[x]` — 完了

## モード中の振る舞い

### タスク着手

1. TODO.md を Read し、`[-]` と重複しないか確認する
2. 該当タスクの `[ ]` を `[-]` に Edit で更新する
3. 作業を開始する

### タスク完了

1. 調査・作業の結果をユーザーに報告する
2. `[x]` には絶対にしない。完了マークはユーザーが自分でつける

### 次のタスクへ

完了後、TODO.md を Read し直して残りの `[ ]` タスクを確認し、上から順に次の `[ ]` タスクに着手する。確認不要。

## ルール

- リストの上から順に作業すること
- 並行して作業しているので、リストは常に更新され続ける。作業着手前に必ず TODO.md を Read し直すこと
- TODO.md の更新には必ず Edit ツールを使う（Bash で sed 等を使わない）
- `[-]` のタスクには手を出さない。他プロセスが作業中
- 1回の呼び出しで扱うタスクは1つだけ
- `[x]` は絶対につけない。完了マークはユーザーが自分でつける
- モード中は常に TODO.md の状態を意識して行動する
